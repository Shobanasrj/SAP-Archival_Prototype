SAP Archiving Assistant

AI-assisted archivability analysis across simulated SAP ECC business objects. Identify cleanup candidates, surface data-quality risk, and prioritize archive runs — with a transparent rules engine plus lightweight ML anomaly detection.

Streamlit pandas · numpy IsolationForest (graceful fallback) Plotly (graceful fallback)
Launch Streamlit app → Read the README

Scan

Simulates table-level scans of FI, MM, SD, CO, PP, and BC archiving objects to detect duplicates, inconsistencies, and incomplete workflows.

Score

Weighted rules engine combines residence time, status, workflow completion, activity recency, legal hold, dependencies, and data quality into a 0–100 archivability score.

Recommend

Each record is bucketed into Archive, Review, Remediate, Deduplicate, or Retain, with a priority score and downloadable executive report.

Run locally on port 8501

From the project root:

cd sap-archiving-assistant
pip install -r requirements.txt
streamlit run app.py --server.port 8501 --server.address 0.0.0.0

Then open http://localhost:8501 — this launcher's Launch Streamlit app button points to the same URL.

What the app shows

  • Executive summary with reclaimable-storage estimate
  • KPI cards: records in scope, archive candidates, review/remediation, data-quality findings
  • Charts: stacked archivability by object, recommendation mix, age vs archivability scatter, recoverable storage by object
  • Per-object inventory and priority worklist (top 50)
  • Object drilldown with rule-by-rule pass/fail breakdown
  • CSV exports (filtered + archive-only) and Markdown executive report

Resilience & portability

  • scikit-learn optional: uses IsolationForest when present, otherwise a deterministic z-score heuristic.
  • Plotly optional: falls back to Streamlit's native chart primitives.
  • No SAP credentials or external services required — entirely self-contained.
